Initialize Constraints as Function of Fixed Effects

Syntax
mixed_object.init_fix_con(fixed_vec)

Private
This cppad_mixed member function is private .

mixed_object
We use mixed_object to denote an object of a class that is derived from the cppad_mixed base class.

fixed_vec
This argument has prototype
     const CppAD::vector<double>& fixed_vec
 It specifies the value of the fixed effects vector @(@ \theta @)@ at which the initialization is done.

init_fix_con_done_
When this function is called, this member variable must be false. Upon return it is true.

fix_con_fun_
On input, the member variable
     CppAD::ADFun<double> fix_con_fun_
 must be empty; i.e., fix_con_fun_.size_var() == 0. If the return value for fix_constraint is empty, fix_con_fun_ is not modified. Otherwise, upon return it contains the corresponding recording for the fix_constraint @(@ c( \theta ) @)@.

fix_con_jac_
The input value of
     CppAD::mixed::sparse_jac_rcv fix_con_jac_
 must be empty. Upon return, fix_con_jac_ contains the sparse_jac_rcv structure for the Jacobian of the constraints .

fix_con_fun_
This ADFun object can be used, with fix_con_jac_, for computing sparse Jacobians; see f .

fix_con_hes_
The input value of
     CppAD::mixed::sparse_hes_rcv fix_con_hes_
 must be empty. If quasi_fixed is false, upon return fix_con_hes_ contains sparse_hes_rcv for the lower triangle of a weighted Hessian for the constraints .

fix_con_fun_
This ADFun object can be used, with fix_con_hes_, for computing sparse Hessians; see f .
