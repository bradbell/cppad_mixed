Private
This class, and all of its members, are implementation details and not part of the CppAD::mixed user API.

References
The values of the arguments are stored by reference and hence the arguments must not be deleted while ipopt_object is still being used.

abort_on_eval_error
This argument has prototype
     const bool& abort_on_eval_error
 If it is true, the fixed effects optimization will abort if an error occurs during the evaluation of one of fixed effects optimizer functions (otherwise it will try to backup).

random_ipopt_options
This argument has prototype
     const std::string& random_ipopt_options
 and is the ipopt_options for optimizing the random effects.

fixed_tolerance
Is the relative convergence criteria used by Ipopt for optimize fixed effects. This only informs ipopt_fixed, the IpoptApplication must be informed separately using
     app->Options()->SetNumericValue("tol", fixed_tolerance)
fixed_lower
This vector has length equal to n_fixed_ and specifies the lower limits for the _effects/cppad_mixed/Fixed Effects, theta/. Note that % - std::numeric_limits<double>::infinity() % is used for minus infinity; i.e., no lower limit.

fixed_upper
This vector has length equal to n_fixed_ and specifies the upper limits for the fixed effects. Note that % std::numeric_limits<double>::infinity() % is used for plus infinity; i.e., no upper limit.

fix_constraint_lower
specifies the lower limits for the constraints . Note that % - std::numeric_limits<double>::infinity() % is used for minus infinity; i.e., no lower limit.

fix_constraint_upper
specifies the upper limits for the constraints. Note that % std::numeric_limits<double>::infinity() % is used for plus infinity; i.e., no upper limit.

fixed_scale
specifies the value of the fixed effects vector @(@ \theta @)@ used for scaling the optimization problem. It must hold for each j that
     fixed_lower[j] <= fixed_scale[j] <= fixed_upper[j]
 The derivative of the objective and constraints at this value for the fixed effects are used to scale the objective and constraint functions. Note that component for which
     fixed_lower[j] == fixed_upper[j]
 are excluded from this scaling.

fixed_in
This vector has length equal to n_fixed_ and specifies the initial value (during optimization) for the fixed effects. It must hold for each j that
     fixed_lower[j] <= fixed_in[j] <= fixed_upper[j]
random_lower
This vector has length equal to n_random_ and specifies the lower limit for the random effects (during optimization).

random_upper
This vector has length equal to n_random_ and specifies the upper limit for the random effects (during optimization).

random_in
This vector has length equal to n_random_ and specifies the initial value (for initial optimization) of the random effects.

mixed_object
The argument mixed_object is an object of a class that is derived from the cppad_mixed base class.

Argument
If name is an argument to the constructor, the variable name_ is a copy of that argument. The variable mixed_object_ is not a copy but rather a reference to mixed_object .

Constants

n_fixed_
is the number of fixed effects.

n_random_
is the number of random effects.

n_fix_con_
is the number of fixed constraints.

n_ran_con
is the number of random constraints.

Temporaries
The following member variables sized by the constructor. They can be used as temporaries, but their sizes should not change.

fixed_tmp_
size n_fixed_

c_vec_tmp_
size n_fix_con_

A_uhat_tmp_
size n_ran_con_

H_beta_tmp_
size n_fixed_

size n_fix_con_

w_laplace_obj_tmp_
size n_ran_con_ + 1.

size fix_likelihood_nabs_ + 1

fix_likelihood_vec_tmp_
If mixed_object_.fix_like_eval returns a vector of size zero, this vector has size zero, otherwise it has size fix_likelihood_nabs_ + 1

Effectively Constant Member Variables
The following member variables are set by the constructor and should not be modified.

nlp_lower_bound_inf_
set to a finite value that is used by Ipopt for minus infinity.

nlp_upper_bound_inf_
set to a finite value that is used by Ipopt for plus infinity.

fix_likelihood_nabs_
number of absolute value terms in the fix_likelihood .

nnz_jac_g_
number of non-zeros in the Jacobian of the ipopt constraints.

nnz_h_lag_
number of non-zeros in the Hessian of the ipopt constraints.

lag_hes_row_
Ipopt row indices for the sparse representation of the Hessian of the Lagrangian (for any Lagrange multiplier values).

lag_hes_col_
Ipopt column indices for the sparse representation of the Hessian of the Lagrangian (for any Lagrange multiplier values).

laplace_obj_hes_2_lag_
Mapping from random object plus constraint Hessian sparse indices to ipopt Hessian sparse indices.

fix_like_hes_2_lag_
Mapping from fixed likelihood Hessian sparse indices to ipopt Hessian sparse indices.

Sparsity Information
The row and col vectors of these sparse_mat_info structures are set by the constructor and assumed to not change. The size of the val vectors is set by the constructor, but element values may change.

ran_con_jac_rcv_
Sparse matrix information for the Jacobian of the random constraints.

laplace_obj_hes_info_
If n_random_ > 0, sparse matrix information for the Hessian of the Laplace objective and constraints.

adaptive_called_
This member variable is set to false.
