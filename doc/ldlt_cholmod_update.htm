Update Factorization Using new Matrix Values

Syntax
ok = ldlt_obj.update(H_rcv)

Prototype 


bool ldlt_cholmod::update(const CppAD::mixed::d_sparse_rcv& H_rcv)

Private
The ldlt_cholmod class is an implementation detail and not part of the CppAD::mixed user API.

Purpose
This routine updates the ldlt_cholmod factorization for new values in the square symmetric matrix.

ldlt_obj
This object has prototype
     CppAD::mixed::ldlt_cholmod ldlt_obj
 In addition, it must have a previous call to ldlt_cholmod_init .

H_rcv
This argument contains new values for the sparse matrix we are computing the LDLT factor of. The sparsity pattern must be the same as in ldlt_cholmod_init . Hence, in particular, it must be in column major order and lower triangular .

sym_matrix_
On input, the member variable
     cholmod_sparse* sym_matrix_
 has been initialized using the sparsity pattern as is in H_rcv . Upon return, values in sym_matrix_ have been set to the corresponding values in the vector H_rcv.val() .

factor_
On input, the member variable
     cholmod_factor* factor_
 has been initialized using the sparsity pattern for the Hessian. Upon return, it contains the factorization
     cholmod_factorize(sym_matrix_, factor_, &common_)
ok
If ok is true, the matrix was factored. Otherwise, the matrix is singular.

Order of Operations
This ldlt_obj function must be called, after the constructor and init and before any other member functions (except pattern ).

Example
The file ldlt_cholmod.cpp contains an example and test that uses this function.
