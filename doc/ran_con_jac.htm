Jacobian of the Random Constraint Function

Syntax
mixed_object.ran_con_jac(
     fixed_vec, random_vec, jac_rcv
)

Private
This cppad_mixed member function is private .

mixed_object
We use mixed_object to denote an object of a class that is derived from the cppad_mixed base class.

fixed_vec
This argument has prototype
     const CppAD::vector<double>& fixed_vec
 It specifies the value of the fixed effects vector @(@ \theta @)@.

random_vec
This argument has prototype
     const CppAD::vector<double>& random_vec
 It specifies the value of the random effects vector @(@ u @)@, This must be the optimal random effects @(@ \hat{u} ( \theta ) @)@.

jac_rcv
This argument has prototype
     sparse_rcv& jac_rcv
 see sparse_rcv .

Sparsity Pattern
In the case where the input value of jac_rcv is empty (all its sizes are zero) only the sparse_rc sparsity pattern in jac_rcv computes. To be specific, upon return jac_rcv.val() has size equal to the number of non-zero elements in the sparsity pattern, but the value of the elements is not specified.

Sparse Matrix
In the case where the input value of jac_rcv is non-empty upon return jac_rcv is a sparse matrix representation of the Jacobian of the random constraint function ; i.e., @[@ \partial_\theta [ A \hat{u} ( \theta ) ] = A \hat{u}_\theta ( \theta ) = - A f_{u,u} \left[ \theta , \hat{u} ( \theta ) \right]^{-1} f_{u,\theta} \left[ \theta , \hat{u} ( \theta ) \right] @]@ see derivative of optimal random effects .

Column Major Order
The results in jac_rcv are in column major order; i.e., upon return
     jac_rcv.col_major[k] = k
 for k = 0, ..., jac_rcv.nnz()-1 .

Example
The file ran_con_jac.cpp contains an example and test of this procedure. It returns true, if the test passes, and false otherwise.
