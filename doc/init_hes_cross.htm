Cross Terms of Sparse Hessian w.r.t Fixed and Random Effects

Syntax
mixed_object.init_hes_cross(
     fixed_vec, random_vec
)

Private
This cppad_mixed member function is private .

mixed_object
We use mixed_object to denote an object of a class that is derived from the cppad_mixed base class.

fixed_vec
This argument has prototype
     const CppAD::vector<double>& fixed_vec
 It specifies the value of the fixed effects vector @(@ \theta @)@ at which the initialization is done.

random_vec
This argument has prototype
     const CppAD::vector<double>& random_vec
 It specifies the value of the random effects vector @(@ u @)@ at which the initialization is done.

hes_cross_
The input value of the member variable
     CppAD::mixed::sparse_hes_rcv hes_cross_
 does not matter. Upon return it contains the sparse_hes_info for the Hessian @[@ f_{u,\theta} ( \theta , u ) @]@ see f(theta, u)

ran_like_fun_, ran_like_a1fun_
Either ran_like_fun_ or ran_like_a1fun_ can be used for the ADFun object in the sparse Hessian Call .

Order
The results are in column major order; i.e.,
     hes_cross_.subset.col()[k] <= hes_cross_.subset.col()[k+1]
     if( hes_cross_.subset.col()[k] == hes_cross_.subset.col()[k+1] )
          hes_cross_.subset.row()[k] < hes_cross_.subset.row()[k+1]
Example
The file hes_cross.cpp contains an example and test of this procedure. It returns true, if the test passes, and false otherwise.
