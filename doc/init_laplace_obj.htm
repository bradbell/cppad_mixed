Second Order Representation of Laplace Objective and Constraints

Syntax
mixed_object.init_laplace_obj(fixed_vec, random_vec)

Private
This cppad_mixed member function is private .

Assumptions
The member variables init_ran_like_done_ and init_newton_checkpoint_done_ are true.

init_laplace_obj_done_
The input value of this member variable must be false. Upon return it is true.

mixed_object
We use mixed_object to denote an object of a class that is derived from the cppad_mixed base class.

fixed_vec
This argument has prototype
     const CppAD::vector<double>& fixed_vec
 It specifies the value of the fixed effects vector @(@ \theta @)@ at which the initialization is done.

random_vec
This argument has prototype
     const CppAD::vector<double>& random_vec
 It specifies the value of the random effects vector @(@ u @)@ at which the initialization is done.

laplace_obj_fun_
The input value of the member variable
     CppAD::ADFun<double> laplace_obj_fun_
 does not matter. Upon return it contains a second order accurate recording of the approximate Laplace objective; see H(beta, theta, u) @(@ H( \beta , \theta , u ) @)@, followed by the approximate random constraint function; see B(beta, theta, u) @(@ B( \beta , \theta , u ) @)@. Thus
     laplace_obj_fun_.Domain() == n_fixed_ + n_fixed_ + n_random_
     laplace_obj_fun_.Range()  == 1 + A_rcv_.nr()
