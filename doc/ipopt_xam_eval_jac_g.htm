Compute Jacobian of Constraint Functions

Syntax
ok = eval_jac_g(
     n, x, new_x, m, nele_jac, iRow, jCol, values
)

n
is the number of variables in the problem (dimension of x).

x
is the value for the primal variables at which the Jacobian of the constraints @(@ \nabla g(x) @)@ is computed (has size n ).

new_x
if true, no Ipopt evaluation method was previous called with the same value for x .

m
is the number of constraints in the problem (dimension of g(x)).

nele_jac
is the number of non-zero elements in the Jacobian of g(x) ; i.e., the same as nnz_jac_g .

iRow
If values is NULL, iRow has size nele_jac and is set to the row indices for the non-zero entries in the Jacobian of the constraints @(@ g_x (x) @)@.

jCol
If values is NULL, jCol has size nele_jac and is set to the column indices for the non-zero entries in the Jacobian of the constraints @(@ g_x (x) @)@.

values
If values is not NULL, it has size nele_jac and values[k] is set to the value of element of the Jacobian @(@ g_x (x) @)@ with row index iRow[k] and column index jCol[k] .

ok
if set to false, the optimization will treat this point like it was not feasible (the function could not be evaluated at this point).

bool ipopt_nlp_xam::eval_jac_g(
     Index           n        ,  // in
     const Number*   x        ,  // in
     bool            new_x    ,  // in
     Index           m        ,  // in
     Index           nele_jac ,  // in
     Index*          iRow     ,  // out
     Index*          jCol     ,  // out
     Number*         values   )  // out
{
     assert( nele_jac == 2 );
     if( values == NULL )
     {
          iRow[0] = 0;
          jCol[0] = 0;
          //
          iRow[1] = 0;
          jCol[1] = 1;
          //
          return true;
     }
     assert( n == 2 );
     assert( m == 1 );
     //
     values[0] = 1.0;
     values[1] = 1.0;
     //
     return true;
}
