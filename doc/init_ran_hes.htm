Initialize Hessian of Random Likelihood w.r.t Random Effects

Syntax
mixed_object.init_ran_hes(
     fixed_vec, random_vec
)

Private
This cppad_mixed member function is private .

Assumptions
The member variable init_ran_like_done_ is true.

init_ran_hes_done_
The input value of this member variable must be false. Upon return it is true.

mixed_object
We use mixed_object to denote an object of a class that is derived from the cppad_mixed base class.

fixed_vec
This argument has prototype
     const CppAD::vector<double>& fixed_vec
 It specifies the value of the fixed effects vector @(@ \theta @)@ at which the initialization is done.

random_vec
This argument has prototype
     const CppAD::vector<double>& random_vec
 It specifies the value of the random effects vector @(@ u @)@ at which the initialization is done.

ran_hes_rcv_
The input value of the member variable
     CppAD::mixed::sparse_rcv ran_hes_rcv_
 does not matter. Upon return it contains the sparse_rcv information for the lower triangle of the Hessian @[@ f_{u,u} ( \theta , u ) @]@ see f(theta, u) The matrix is symmetric and hence can be recovered from its lower triangle.

ran_hes_work_
The input value of the member variable
     CppAD::sparse_hes_work ran_hes_work_
 does not matter (it should be empty). Upon return, it contains the necessary information for the calls to sparse_hes specified below.

ran_like_fun_
If the return value from ran_likelihood_hes is non-empty, and NDEBUG is not defined, ran_likelihood_hes is checked for correctness. If the return value from ran_likelihood_hes is empty, upon return the following call can be used to compute the lower triangle of the Hessian @(@ f_{u,u} ( \theta , u ) @)@ as a d_vector:
     ran_like_fun_.sparse_hes(
          x,
          w
          ran_hes_rcv_,
          not_used_pattern,
          not_used_coloring,
          ran_hes_work_
     )
 where the values of not_used_pattern and not_used_coloring do not matter.

Random Effects Index
The indices in ran_hes_rcv_ are relative to both the fixed and random effects with the fixed effects coming first. To get the indices relative to just the random effects, subtract n_fixed_; e.g, the value
     ran_hes_rcv_.val()[k]
 corresponds the second partial with respect to the following two random effect indices:
     ran_hes_rcv_.row()[k] - n_fixed_
     ran_hes_rcv_.col()[k] - n_fixed_
Order
The results are in column major order; i.e.,
     ran_hes_rcv_.col()[k] <= ran_hes_rcv_.col()[k+1]
     if( ran_hes_rcv_.col()[k] == ran_hes_rcv_.col()[k+1] )
          ran_hes_rcv_.row()[k] < ran_hes_rcv_.row()[k+1]
ran_hes_fun_
The input value of the member variables
     CppAD::ADFun<double> ran_hes_fun_
 does not matter. Upon return its zero order forward mode computes the lower triangle of the sparse Hessian @[@ f_{u,u} ( \theta , u ) @]@ in the same order as the elements of ran_hes_rcv_ (and a1_hes_rcv_).
